# vibe kanban用ワークフロー

以下の手順に従う

1. 設計書の準備
2. 実装とレビューのループ
3. 変更のコミット(1回目)
4. リファクタリング
5. 変更のコミット(2回目)
6. レポートを出力する

## 設計書の準備

`vibes/tmp/{ブランチ名}/design.md` に指示された「design」を出力してください


## 実装とレビューのループ

以下を繰り返す

1. 実装/レビュー対応： `vibes/kanban/simple-impl.md` に従う
2. レビュー： `./vibes/kanban/review-claude.sh` を実行する
  - MUST: timeout は 500000ms とする
  - MUST: API リミットで失敗した場合、継続不能なのでエラーとする
3. レビューチェック：
  - `vibes/tmp/{ブランチ名}/review.md` を読んで、そのレビューが妥当かどうかを判断する
  - 指摘事項がなくなった・妥当なレビューがなくなった場合、このループを終了し次に移る

## 変更のコミット(1回目)

- 変更のgit commitを行う
- これは「単純な変更まで」の変更を保存するためである
- gitignoreの対象は含めないこと

## リファクタリング

- `vibes/kanban/refactor.md` に従う

## 変更のコミット(2回目)

- 変更のgit commitを行う
- これは「最終的な変更まで」の変更を保存するためである
- gitignoreの対象は含めないこと

## 最終レポートを出力する

- ほかに回しす予定のタスクなど、レポートを出力する
